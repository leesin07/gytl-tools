# 🚀 部署前准备清单

## 第一阶段：服务器准备

### ☐ 1. 服务器选购
- [ ] 已购买云服务器（推荐配置：2核4GB，40GB SSD）
- [ ] 已选择操作系统（Ubuntu 20.04/22.04）
- [ ] 已确认服务器所在地区（建议选择离用户近的地区）

### ☐ 2. 网络配置
- [ ] 已获取服务器公网 IP 地址
- [ ] 已配置安全组规则（开放端口：22、80、443）
- [ ] 已测试服务器可正常访问（SSH 连接）

---

## 第二阶段：域名准备

### ☐ 3. 域名购买
- [ ] 已购买域名（阿里云、腾讯云或 Cloudflare）
- [ ] 已准备域名管理账号和密码

### ☐ 4. 域名解析
- [ ] 已登录域名管理控制台
- [ ] 已添加 A 记录（`@` → 服务器 IP）
- [ ] 已添加 A 记录（`www` → 服务器 IP）
- [ ] 已确认 DNS 生效（等待 10-30 分钟）

---

## 第三阶段：代码准备

### ☐ 5. 代码仓库
- [ ] 已将代码推送到 GitHub/GitLab
- [ ] 已确认仓库地址可访问

### ☐ 6. 环境变量
- [ ] 已复制 `.env.example` 为 `.env`
- [ ] 已根据需要修改环境变量配置
- [ ] 已检查 `.env` 文件格式是否正确

---

## 第四阶段：本地测试

### ☐ 7. 本地构建测试
- [ ] 已在本地执行 `pnpm install`
- [ ] 已在本地执行 `pnpm run build`
- [ ] 已在本地执行 `pnpm start`
- [ ] 已在本地访问 `http://localhost:3000` 确认正常运行
- [ ] 已测试核心功能（股票筛选、搜索等）

### ☐ 8. Docker 镜像测试
- [ ] 已在本地执行 `docker build -t gytl-tools:latest .`
- [ ] 已在本地执行 `docker run -p 3000:3000 gytl-tools:latest`
- [ ] 已在本地访问 `http://localhost:3000` 确认正常运行
- [ ] 已停止并删除测试容器

## 📝 部署时间估算

| 阶段 | 预计时间 | 说明 |
|------|----------|------|
| 服务器初始化 | 10-15 分钟 | 安装 Docker、配置系统 |
| 项目部署 | 20-30 分钟 | 克隆代码、构建镜像、启动容器 |
| 域名配置 | 10-30 分钟 | DNS 解析生效时间 |
| SSL 证书 | 5-10 分钟 | Certbot 自动配置 |
| Nginx 配置 | 10-15 分钟 | 反向代理配置 |
| 监控配置 | 10-15 分钟 | 日志、告警配置 |
| **总计** | **65-115 分钟** | 约 1-2 小时 |

## 🆘 常见问题预查

### 如果遇到以下问题，请提前准备好解决方案：

1. **Docker 安装失败**
   - 准备备用安装方法：使用脚本手动安装

2. **DNS 解析不生效**
   - 确认已等待足够时间（最多 48 小时）
   - 准备临时使用 IP 访问

3. **SSL 证书获取失败**
   - 准备手动配置 SSL 证书
   - 或暂时使用 HTTP 访问

4. **容器启动失败**
   - 准备查看日志的方法
   - 准备重新构建镜像

5. **Nginx 配置错误**
   - 准备测试 Nginx 配置命令
   - 准备恢复默认配置
