# GYTL-Tools 工具脚本使用指南

> 本文档介绍如何使用项目中的自动化脚本来简化部署和测试流程。

---

## 📋 可用脚本

| 脚本名称 | 用途 | 使用场景 |
|---------|------|---------|
| `quick-deploy.sh` | 快速部署 | 日常更新到生产环境 |
| `pre-deploy-test.sh` | 部署前测试 | 部署前验证代码质量 |
| `quick-rollback.sh` | 快速回滚 | 部署失败或出现问题时回滚 |
| `test-api.js` | API 连接测试 | 测试东方财富 API 连接 |
| `build.sh` | 构建项目 | 本地构建项目 |
| `dev.sh` | 启动开发服务器 | 本地开发测试 |
| `start.sh` | 启动生产服务器 | 启动生产环境服务 |

## 🚀 快速部署脚本

### 用途

快速将代码部署到生产环境，包含以下步骤：

1. 拉取最新代码
2. 创建备份
3. 安装依赖
4. TypeScript 类型检查
5. 构建项目
6. 重新加载 PM2 服务

### 使用方法

```bash
# 进入项目目录
cd /workspace/projects

# 运行快速部署脚本
./scripts/quick-deploy.sh
```

## ⏪ 快速回滚脚本

1. 显示最近的提交历史
2. 选择要回滚的版本
3. 停止当前服务
4. 回滚代码
5. 重新构建和启动服务

### 使用方法

```bash
# 进入项目目录
cd /workspace/projects

# 运行快速回滚脚本
./scripts/quick-rollback.sh
```

## 📝 推荐的工作流程

### 日常更新流程

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 部署前测试
./scripts/pre-deploy-test.sh

# 3. 如果测试通过，快速部署
./scripts/quick-deploy.sh

# 4. 验证部署结果
curl -I http://localhost:5000
```

### 紧急修复流程

```bash
# 1. 快速修复问题
vim src/app/page.tsx

# 2. 测试修复
./scripts/pre-deploy-test.sh

# 3. 部署修复
./scripts/quick-deploy.sh
```

### 出现问题回滚流程

```bash
# 1. 快速回滚
./scripts/quick-rollback.sh

# 2. 验证回滚
curl -I http://localhost:5000

# 3. 查看服务日志
pm2 logs gytl-tools --lines 50
```

## ⚠️ 注意事项

1. **权限问题**：所有脚本都有执行权限，如果提示权限错误，运行：
   ```bash
   chmod +x scripts/*.sh
   ```

2. **环境要求**：确保已安装 Node.js 24.x 和 pnpm 9.x

3. **PM2 配置**：确保 PM2 已正确配置，服务名称为 `gytl-tools`

4. **备份策略**：建议定期备份 `/workspace/backups/` 目录

5. **日志查看**：部署后查看 PM2 日志确认服务正常：
   ```bash
   pm2 logs gytl-tools --lines 50
   ```

## 🆘 常见问题

### Q1: 脚本执行提示 "Permission denied"

**A:** 添加执行权限
```bash
chmod +x scripts/*.sh
```

### Q2: 部署后服务无法访问

**A:** 检查服务状态和日志
```bash
pm2 status
pm2 logs gytl-tools --lines 50
```

### Q3: 回滚后数据丢失

**A:** 回滚只回退代码，不回退数据。如需数据恢复，使用备份：
```bash
# 恢复最新的备份
cp -r /workspace/backups/gytl-tools-*/.next ./
```

### Q4: 测试脚本一直失败

**A:** 检查开发服务器是否正常启动
```bash
# 手动启动开发服务器
pnpm dev

# 检查端口
lsof -i:5000
```
